REPOSITORY : PurchaseRepository
FICHIER : src/Repositories/PurchaseRepository.php
TABLE SQL : `purchase_history`

MÉTHODES DÉTAILLÉES :

1. add($userId, $itemId, $itemName, $price)
   - Rôle : Enregistre définitivement une transaction.
   - Données stockées : L'ID du joueur, l'ID du produit, le nom de l'item (pour garder une trace même si l'item est supprimé plus tard) et le prix payé.

2. getByUserId($userId)
   - Paramètre : ID du joueur.
   - Retour : Liste des achats classés du plus récent au plus ancien.
   - Utilisation : Affiche l'historique personnel sur la page de profil du joueur.

IMPORTANCE :
Cette classe est critique car elle constitue la preuve de transaction pour les joueurs. Elle est liée à la table `user` via une clé étrangère avec suppression en cascade.
