CLASSE : Controller (Base)
FICHIER : src/Controllers/Controller.php

DESCRIPTION :
C'est la classe mère de tous les contrôleurs du site. Elle fournit les outils nécessaires pour gérer l'affichage et les redirections.

MÉTHODES DÉTAILLÉES :

1. render($view, $data = [])
   - Paramètres : `$view` (le nom du fichier) et `$data` (les variables à passer).
   - Technique : Utilise la capture de flux (`ob_start`) pour injecter le contenu d'une page dans un gabarit global (`layout.php`).
   ```php
   extract($data); // Transforme ['msg' => 'Hi'] en $msg
   ob_start();
   require_once 'pages/' . $view . '.php'; // Charge la vue
   $content = ob_get_clean(); // Capture le HTML
   require_once 'layout.php'; // Affiche dans le gabarit
   ```

2. redirect($url)
   - Rôle : Envoie un en-tête HTTP de redirection et arrête le script immédiatement.

PRINCIPE : L'ENCAPSULATION (LAYOUT)
-----------------------------------
Plutôt que de recopier le Header et le Footer sur chaque page, on utilise un **Layout unique**. 
Le contrôleur "capture" le code HTML de la page spécifique et l'injecte dans la variable `$content` du layout. Cela permet de changer le design de TOUT le site en modifiant un seul fichier.
C'est le principe fondamental du DRY (Don't Repeat Yourself).
