REPOSITORY : ProductRepository
FICHIER : src/Repositories/ProductRepository.php
TABLE SQL : `boutique`

MÉTHODES DÉTAILLÉES :

1. save($data, $id = null)
   - Rôle : Enregistre ou modifie un produit de la boutique.
   - Gestion d'image : Identique au repository d'articles (ne remplace pas l'image si aucune nouvelle n'est envoyée).

2. updateQuantity($id, $newQuantity)
   - Rôle : Met à jour directement le stock d'un produit spécifique.

3. reduceStock($id, $quantity)
   - Rôle : Soustrait automatiquement X unités du stock lors d'une transaction.
   ```sql
   UPDATE boutique SET quantite = quantite - ? WHERE id = ?
   ```

STRUCTURE SQL :
- id : Identifiant unique.
- titre : Nom de l'objet.
- prix : Valeur en crédits joueurs.
- description : Détails sur l'item.
- image : Nom du fichier image.

PRINCIPE : GESTION DU CATALOGUE
-------------------------------
Les produits sont les objets centraux de l'économie. Bien que le stock soit actuellement illimité par défaut dans la logique d'achat, le Repository est déjà prêt pour une gestion de **quantités limitées**.
Chaque produit possède un prix en **Crédits**, qui est la monnaie virtuelle unique du serveur.
