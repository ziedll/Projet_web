CONTRÔLEUR : AdminController
FICHIER : src/Controllers/AdminController.php

MÉTHODES DÉTAILLÉES :

1. __construct()
   - Sécurité : `Auth::requireAdmin()`.
   - Rôle : Bloque l'accès à TOUTES les méthodes de ce contrôleur si l'utilisateur n'est pas un administrateur.

2. index()
   - Rôle : Affiche le dashboard d'administration.
   - Paramètres GET : `section` (articles, boutique, users) et `edit` (ID de l'élément à modifier).
   - Logique : Charge toutes les données nécessaires pour les listes et, si un `edit` est demandé, charge l'élément spécifique.

3. handleArticle() / handleProduct() / handleUser()
   - Rôle : Traitement des formulaires POST.
   - Code commun :
   ```php
   $image = $this->handleUpload($_FILES['image']);
   if ($image) $data['image'] = $image;
   $this->repo->save($data, $id);
   ```

4. handleUpload($file) (Privé)
   - Paramètre : Le fichier provenant de `$_FILES`.
   - Rôle : Utilitaire de sécurité. Vérifie l'erreur, crée le dossier `uploads/` si besoin, génère un nom de fichier unique (`uniqid`) et déplace le fichier sur le disque.

PRINCIPE : L'ADMINISTRATION CENTRALISÉE
---------------------------------------
L'administration utilise un contrôleur unique pour gérer plusieurs entités afin d'éviter la duplication de code pour des fonctions similaires (CRUD).
Le système repose sur des **redirections avec messages flash** (`admin?msg=Succès`) pour informer l'administrateur de la réussite de ses actions sans avoir besoin de rafraîchir manuellement la page.
C'est ici que se trouve le "cerveau" de gestion du contenu du serveur.
