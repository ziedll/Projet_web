CONTRÔLEUR : BoutiqueController
FICHIER : src/Controllers/BoutiqueController.php

MÉTHODES DÉTAILLÉES :

1. index()
   - Rôle : Affiche la boutique.
   - Logique : Récupère la liste des produits (`ProductRepository`) et le solde de l'utilisateur s'il est connecté.

2. buy()
   - Rôle : Traite la transaction d'achat.
   - Sécurité : `Auth::requireLogin()` obligatoire. Vérifie si le solde est suffisant.
   ```php
   if ($product && $userData['credit'] >= $product['prix']) {
       $userRepo->updateCredit($user['id'], -$product['prix']); // Débit
       $purchaseRepo->add($userId, $itemId, $product['titre'], $product['prix']); // Historique
   }
   ```

PRINCIPE : TRANSACTION SÉCURISÉE
--------------------------------
L'achat ne se contente pas de vérifier le prix, il effectue trois actions critiques en une seule fois :
1. **Vérification** du solde en temps réel (côté serveur).
2. **Débit** immédiat des crédits en base de données.
3. **Enregistrement immuable** de la transaction (Snapshot) pour l'historique.
Cela garantit qu'un joueur ne peut pas "tricher" sur son solde en manipulant le navigateur.
