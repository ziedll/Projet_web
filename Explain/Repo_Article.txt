REPOSITORY : ArticleRepository
FICHIER : src/Repositories/ArticleRepository.php
TABLE SQL : `article`

MÉTHODES DÉTAILLÉES :

1. getAll() (Surchargé)
   - Rôle : Récupère tous les articles du plus récent au plus ancien (`ORDER BY date_de_creation DESC`).

2. getLatest($limit = 3)
   - Paramètre : Le nombre d'articles à récupérer.
   - Rôle : Utilisé sur la page d'accueil pour n'afficher qu'un aperçu des dernières news.
   ```sql
   SELECT * FROM article ORDER BY date_de_creation DESC LIMIT 3
   ```

3. save($data, $id = null)
   - Paramètres : `$data` (Données du formulaire), `$id` (ID de l'article si mise à jour).
   - Rôle : Méthode "intelligente" pour Créer ou Modifier.
   - Logique :
     - Si `$id` est fourni : Exécute une requête `UPDATE`. Gère la modification optionnelle de l'image.
     - Si `$id` est `null` : Exécute une requête `INSERT` pour créer un nouvel article.

PRINCIPE : SÉPARATION DU CONTENU ET DU CODE
-------------------------------------------
Plutôt que d'écrire du texte directement dans les pages HTML, nous utilisons ce Repository pour transformer les lignes de la base de données SQL en objets PHP utilisables.
Cela permet de changer le design du site (CSS) sans jamais toucher aux articles eux-mêmes. C'est le cœur du système d'actualités dynamique du projet.
