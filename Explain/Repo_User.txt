REPOSITORY : UserRepository
FICHIER : src/Repositories/UserRepository.php
TABLE SQL : `user`

MÉTHODES DÉTAILLÉES :

1. getByEmail($email)
   - Rôle : Utilisée par le système de connexion pour trouver le compte lié à un email.

2. create($data)
   - Rôle : Crée un nouveau compte (Inscription). Hashage automatique des mots de passe géré en amont par le contrôleur.

3. updateCredit($id, $amount)
   - Paramètres : `$id` de l'utilisateur, `$amount` (montant positif pour ajouter, négatif pour retirer).
   - Rôle : Modifie le solde de l'utilisateur de manière sécurisée en base de données.
   ```php
   public function updateCredit($id, $amount) {
       $stmt = $this->db->prepare("UPDATE user SET credit = credit + ? WHERE id = ?");
       return $stmt->execute([$amount, $id]);
   }
   ```

4. updateProfile($id, $data)
   - Rôle : Gère l'Espace Joueur (Extranet). Met à jour l'avatar, la description et les liens sociaux.
   - Astuce SQL : Utilise `COALESCE(?, avatar)` pour ne pas écraser l'avatar si aucun nouveau fichier n'est envoyé.
   ```sql
   UPDATE user SET avatar = COALESCE(?, avatar), description = ? WHERE id = ?
   ```

5. getFreshData($id)
   - Rôle : Récupère uniquement le rôle et les crédits. Permet de mettre à jour la session PHP en temps réel sans recharger toutes les informations lourdes.

PRINCIPE : L'ÉCONOMIE DU JOUEUR
-------------------------------
Les crédits sont gérés directement en SQL (`credit = credit + ?`) pour éviter les erreurs de calcul PHP et les problèmes de concurrence. C'est la méthode la plus fiable pour une monnaie virtuelle.
Toute modification de profil importante doit synchroniser la session PHP pour que le joueur voie ses changements immédiatement.
